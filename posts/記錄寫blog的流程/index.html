[記錄]寫Blog的流程

本文記錄我寫Blog的流程。自從平台轉至Hugo後，寫文章的流程大部分都可以用 Github desktop, Visual studio code 完成。感覺上整個流程更集中，可以更專心，不用不停轉換程式去寫一篇文，大部分的工作都可以在 VS code 內完成 (IDE 的好處!) 。

流程

  1. 開啟 Github desktop
  2. 選擇 blog 的 Repo
  3. 開啟 VS code (在 Github desktop 設定 VS code 為預設程式)
  4. 在 VS code 內新增 Markdown (可以不用啟動 Hugo) 在 VS code 的 explorer 中，移至 content\posts folder 內新增一個 markdown file (*.md)
  5. post 開頭的 title, date, draft 可以從之前的文章 copy 後更改
  6. 文章就是用 markdown 語法書寫
  7. VS code 有很多 markdown 的 plugin
  8. 專心寫文
  9. 以下會開啟 Hugo 作 blog 的 rendering
  10. 在 VS code 打開 cmd
  11. 在 cmd 打開 hugo-p.bat
  12. 移至 hugo blog 的 folder，cd mattkoinuk-sources
  13. 啟動 hugo server， hugo server -D
  14. 註意 -D 的參數代表 draft 的文章也會顯示，如果沒有設定，draft 文章不會顯示
  15. 在 cmd 的 return 中有 hugo server 的接口位置，我的是 http://localhost:1313/
  16. 在 brower 中開啟，作文章的預覽及更正
  17. 直接在 VS code 內更改即可，hugo 會自動更新，不需 F5
  18. 在 cmd 內按 Ctrl+C 停止 hugo server
  19. 在 cmd 內輸入 hugo， 程式會將結果輸出至 public folder 內
  20. 關閉 cmd
  21. 在 Github desktop 中，移至 mattkoinuk-sources Repo ，commit change， push to github
  22. 在 Github desktop 中，移至 mattkoinuk.github.io Repo ，commit change， push to github
  23. 可在網站看到文章。
  24. ❤️
