[記錄]寫Blog的流程

[記錄]寫Blog的流程 本文記錄我寫Blog的流程。自從平台轉至Hugo後，寫文章的流程大部分都可以用 Github desktop, Visual studio code 完成。感覺上整個流程更集中，可以更專心，不用不停轉換程式去寫一篇文，大部分的工作都可以在 VS code 內完成 (IDE 的好處!) 。 流程 開啟 Github desktop 選擇 blog 的 Repo 開啟 VS code (在 Github desktop 設定 VS code 為預設程式) 在 VS code 內新增 Markdown (可以不用啟動 Hugo) 在 VS code 的 explorer 中，移至 content\posts folder 內新增一個 markdown file (*.md) post 開頭的 title, date, draft 可以從之前的文章 copy 後更改 文章就是用 markdown 語法書寫 VS code 有很多 markdown 的 plugin 專心寫文 以下會開啟 Hugo 作 blog 的 rendering 在 VS code 打開 cmd 在 cmd 打開 hugo-p.
